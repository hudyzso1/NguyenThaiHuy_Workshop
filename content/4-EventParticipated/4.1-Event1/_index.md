- **Community Connection:** Create an environment for networking, knowledge sharing, and exchanging practical experience among AWS learners and technology professionals.

- **Khắc Uy Phạm:** Discussed behavioral psychology and methods for overcoming procrastination.

---

### Key Presentation Highlights

#### 1. Improving AWS Skills with Cloud Quest and Floci

The presentation introduced alternative methods for learning and practicing AWS without creating unnecessary cloud costs.

- **AWS Cloud Quest:** An interactive learning platform where users complete cloud-related tasks through practical scenarios.
- **Floci:** An open-source AWS emulator that allows developers to test AWS services locally.
- Floci helps reduce the risk of unexpected costs during the learning process.
- Local testing allows developers to experiment before deploying resources to a real AWS environment.

The recommended learning process includes:

1. Learn basic AWS concepts through AWS Cloud Quest.
2. Build and test cloud applications locally using Floci.
3. Deploy the completed application to the AWS Cloud environment.

#### 2. Hackathon Experience

Team Khương shared their experience participating in a 36-hour hackathon and developing the UTMorpho project.

UTMorpho is an AI-based application that supports real-time interface generation and editing through a visual canvas.

The presentation also introduced other projects:

- **SynthHunter:** An AI-based voice authentication system for detecting deepfake audio.
- **Vortex:** An automated recruitment platform designed to reduce candidate interview anxiety.

#### 3. Building Confidence in Technology

The session discussed the importance of communication and confidence in the technology industry.

Important lessons included:

- Confidence does not mean knowing every answer.
- Confidence is the willingness to take action and accept responsibility.
- Communication skills are important when presenting technical ideas.
- The five-second rule can help reduce hesitation and encourage immediate action.

#### 4. AI Agents and AWS Architecture

The “Tử Vi Đại Việt” project demonstrated how AWS and AI services can be combined to create a scalable cloud application.

The system architecture included:

- Amazon Bedrock
- AWS Lambda
- Amazon API Gateway
- Amazon DynamoDB
- Amazon S3
- Amazon CloudFront

The serverless architecture helps the system automatically scale based on user traffic while reducing infrastructure management and operating costs.

#### 5. DevOps Mindset and Behavioral Psychology

The DevOps presentation emphasized that DevOps is not only about tools and automation.

DevOps also includes:

- People
- Processes
- Communication
- Collaboration
- Organizational culture

The session about procrastination explained that procrastination may result from fear of failure, pressure, or emotional discomfort.

The suggested solution was the **five-minute rule**, which involves dividing a large task into a small action that can be started immediately.

### Key Takeaways

#### Mindset and Soft Skills

- Technical knowledge should be combined with communication and presentation skills.
- Complex technical topics should be explained clearly and logically.
- Confidence develops through continuous practice and experience.
- Large objectives should be divided into smaller and manageable tasks.
- Learning from real projects provides a better understanding of technology.

#### Technical Knowledge

- AWS serverless services can help build scalable and cost-effective applications.
- Amazon Bedrock can be integrated with AWS services to develop AI-powered systems.
- Local AWS emulators can reduce costs during development and testing.
- DevOps focuses on collaboration, processes, automation, and continuous improvement.
- Cloud architecture should consider security, scalability, performance, and cost optimization.
